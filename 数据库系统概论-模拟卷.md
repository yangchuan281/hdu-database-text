## 二、分析题（共37分）

### 1. SQL 语句操作改错题（6分）

已知表结构：
```
学生(学号 CHAR(8), 姓名 VARCHAR(20), 性别 CHAR(2), 年龄 INT, 院系 VARCHAR(30))
选课(学号 CHAR(8), 课程号 CHAR(4), 成绩 INT)
课程(课程号 CHAR(4), 课程名 VARCHAR(30), 学分 INT, 先行课 CHAR(4))
```

以下各 SQL 语句存在错误，请找出并改正。

**(1)**（2分）查询所有学生的姓名和年龄，结果按年龄降序排列：
```sql
SELECT 姓名, 年龄 FROM 学生 ORDER BY 年龄;
```

**(2)**（2分）查询各院系的平均年龄：
```sql
SELECT 院系, AVG(年龄) FROM 学生;
```

**(3)**（2分）查询选修了课程号为'C001'的课程的学生姓名：
```sql
SELECT 姓名 FROM 学生, 选课 WHERE 学号 = 选课.学号 OR 课程号 = 'C001';
```

### 2. 规范化（8分）

函数依赖集 F = {
教师号 → 教师名, 职称,
课程号 → 课程名, 课时,
(教师号, 课程号) → 成绩
}

**(1)**（2分）求 R 的候选码。

**(2)**（2分）判断 R 属于第几范式，说明理由。

**(3)**（4分）将 R 分解到满足 3NF（写出分解结果，并标明主码和外码）。

### 3. 并发控制（8分）

设有两个事务：
```
T1: READ(A); A=A-200; WRITE(A); READ(B); B=B+200; WRITE(B);
T2: READ(A); A=A*1.2; WRITE(A);
```

初始值：A=1000, B=500

**(1)**（4分）按以下调度执行，给出最终 A 和 B 的值，并判断是否可串行化：
```
T1: READ(A)=1000 → T1: A←800 → T1: WRITE(A)=800
→ T2: READ(A)=800 → T2: A←960 → T2: WRITE(A)=960
→ T1: READ(B)=500 → T1: B←700 → T1: WRITE(B)=700
```

**(2)**（4分）如果使用封锁机制，说明共享锁（S锁）和排他锁（X锁）的区别，并解释二段锁协议的基本思想。

### 4. 故障恢复（10分）

```

(T2, BEGIN)
(T1, B, 100, 150)
(T1, COMMIT)
(T2, C, 200, 180)
(T3, BEGIN)
(T3, D, 300, 250)
——— 系统故障 ———
```

**(1)**（4分）系统故障发生后，需要对哪些事务执行 UNDO？哪些执行 REDO？说明理由。

**(2)**（3分）分别写出 UNDO 和 REDO 的具体操作内容。

**(3)**（3分）如果发生的是介质故障（磁盘损坏），恢复策略与系统故障有何不同？需要借助什么额外的条件？
