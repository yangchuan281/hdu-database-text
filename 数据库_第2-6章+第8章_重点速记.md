# 📘 第2章：关系数据库（考试分值最高）

## 一、关系模型三要素

| 要素 | 说明 |
|:----|:------|
| **关系数据结构** | 单一结构——**二维表**（关系），实体和联系都用表表示 |
| **关系操作** | 关系代数 + SQL，**一次一集合**，存取路径对用户透明 |
| **关系完整性约束** | 实体完整性 + 参照完整性 + 用户定义完整性 |

## 二、基本术语

| 术语 | 含义 | 例子 |
|:----|:-----|:-----|
| **关系** | 一张二维表 | Student表 |
| **元组** | 行（记录） | 表中的一行数据 |
| **属性** | 列（字段） | 学号、姓名 |
| **码（Key）** | 唯一标识元组的属性集 | 学号 |
| **候选码** | 能唯一标识元组的所有最小属性集 | (学号)、(身份证号) |
| **主码** | 选中的一个候选码 | 学号 |
| **外码** | 本表属性，引用另一表的主码 | 选课表中的学号 |
| **域** | 属性的取值范围 | 性别∈{男,女} |
| **分量** | 元组中的一个属性值 | 某个单元格 |
| **关系模式** | 关系名(属性名集合) | 学生(学号,姓名,性别) |

## 三、关系代数 ★★★★★（编程题12分）

### 5种基本运算（必须背）

| 运算 | 符号 | 作用 | 类比SQL |
|:----|:----:|:-----|:--------|
| **选择** | **σ** | 选**行**（满足条件） | WHERE |
| **投影** | **π** | 选**列**（指定属性） | SELECT |
| **笛卡尔积** | **×** | 两表所有行两两组合 | FROM A, B |
| **并** | **∪** | 上下拼接（同结构） | UNION |
| **差** | **−** | 上减下（同结构） | EXCEPT / NOT IN |

### 3种导出运算

| 运算 | 符号 | 本质公式 | 场景 |
|:----|:----:|:---------|:-----|
| **交** | ∩ | A∩B = A−(A−B) | 两表共有 |
| **连接** | ⋈ | σ条件(A×B) | **跨表查询（最重要！）** |
| **除** | ÷ | 见下方 | **"至少包含了全部……"** |

### 连接详解

```
自然连接：A ⋈ B → 按同名属性自动匹配
等值连接：A ⋈<sub>A.key=B.key</sub> B → 按指定条件匹配
```

### 除运算（最抽象）

> **R ÷ S = 找出 R 中包含了 S 里全部项目的记录**

**典型考题：** "选修了全部课程的学生学号"
```
π<sub>学号,课程号</sub>(选课) ÷ π<sub>课程号</sub>(课程)
```

**做题模板：**
| 问题 | 模板 |
|:----|:-----|
| 选某条件的行 | σ<sub>条件</sub>(表) |
| 只要某些列 | π<sub>列名</sub>(表) |
| 跨表查询 | 表1 ⋈ 表2 ⋈ 表3 |
| "全部""所有" | R ÷ S |

# 📘 第3章：SQL语言（~30分）

## 一、SQL四大分类

| 分类 | 命令 | 用途 |
|:----|:-----|:-----|
| **DDL** | CREATE / ALTER / DROP | 定义数据结构 |
| **DML** | SELECT / INSERT / UPDATE / DELETE | 操作数据 |
| **DCL** | GRANT / REVOKE | 权限控制 |
| **TCL** | COMMIT / ROLLBACK | 事务控制 |

## 二、SELECT 万能模板

```sql
SELECT 列名                  -- π（投影）
FROM 表名                    -- 数据来源
WHERE 条件                   -- σ（选择）
GROUP BY 分组列              -- 分组
HAVING 分组后条件             -- 对分组结果过滤
ORDER BY 排列表 ASC|DESC;    -- 排序
```

## 三、常考查询模式

### ① 单表查询
```sql
SELECT 姓名, 年龄 FROM 学生 WHERE 院系 = 'CS';
```

### ② 多表连接查询
```sql
SELECT 学生.姓名, 选课.成绩
FROM 学生, 选课
WHERE 学生.学号 = 选课.学号 AND 选课.课程号 = 'C01';
```

### ③ 分组统计
```sql
SELECT 院系, COUNT(*) AS 人数, AVG(年龄) AS 平均年龄
FROM 学生
GROUP BY 院系;
```

### ④ 嵌套查询
```sql
SELECT 姓名 FROM 学生 WHERE 学号 IN (
  SELECT 学号 FROM 选课 WHERE 课程号 = 'C01'
);
```

### ⑤ 带 EXISTS 的查询（比 IN 强大）
```sql
-- 查询选修了'C01'课程的学生
SELECT 姓名 FROM 学生 WHERE EXISTS (
  SELECT * FROM 选课 WHERE 学号 = 学生.学号 AND 课程号 = 'C01'
);
```

### ⑥ HAVING 过滤分组
```sql
-- 查询选修课程数≥3门的学生
SELECT 学号, COUNT(*) FROM 选课
GROUP BY 学号
HAVING COUNT(*) >= 3;
```

## 四、DDL

```sql
CREATE TABLE 学生 (
    学号 CHAR(8) PRIMARY KEY,        -- 实体完整性
    姓名 VARCHAR(20) NOT NULL,
    性别 CHAR(2) CHECK(性别 IN('男','女')), -- 用户定义完整性
    院系 VARCHAR(30),
    班级号 CHAR(6) REFERENCES 班级(班级号)  -- 参照完整性
);

ALTER TABLE 学生 ADD 年龄 INT;
DROP TABLE 学生;

CREATE VIEW 计算机系学生 AS
SELECT 学号, 姓名 FROM 学生 WHERE 院系 = 'CS';

CREATE INDEX idx_姓名 ON 学生(姓名);
```

## 五、DCL

```sql
GRANT SELECT, INSERT ON 学生 TO 张三;
GRANT ALL ON 学生 TO 李四 WITH GRANT OPTION;  -- 可转授权
REVOKE INSERT ON 学生 FROM 张三;

-- 角色管理
CREATE ROLE 教师;
GRANT SELECT ON 学生 TO 教师;
GRANT 教师 TO 王五;
```

# 📘 第4章：数据库安全性 + 完整性

## 一、安全性

| 手段 | 说明 |
|:----|:------|
| **用户身份鉴别** | 密码、指纹、智能卡 |
| **存取控制** | DAC（自主）/ MAC（强制） |
| **视图机制** | 隐藏敏感数据 |
| **审计** | 记录操作日志 |
| **数据加密** | 加密存储/传输 |

## 二、三类完整性 ★★★

| 完整性 | 含义 | SQL实现 |
|:-------|:-----|:--------|
| **实体完整性** | 主码不能为空 | `PRIMARY KEY(学号)` |
| **参照完整性** | 外码必匹配主码或为空 | `FOREIGN KEY(班级号) REFERENCES 班级(班级号)` |
| **用户定义完整性** | 自定义规则 | `CHECK(年龄>0 AND 年龄<150)` |

## 三、触发器（选择题）

```sql
-- 自动触发：删除学生时检查是否有未还书
CREATE TRIGGER 检查借书
ON 学生 AFTER DELETE
AS
    IF EXISTS(SELECT * FROM 借阅 WHERE 学号 IN (SELECT 学号 FROM deleted))
        ROLLBACK;
```

**💡 安全性防坏人，完整性防错误**

# 📘 第5章：关系规范化理论（分析题8分）

## 一、函数依赖

| 类型 | 定义 | 例子 |
|:----|:-----|:-----|
| **完全函数依赖** | 需要完整候选码才能决定 | (学号,课程号)→成绩 |
| **部分函数依赖** | 候选码的一部分就能决定 | (学号,课程号)→姓名（学号→姓名）|
| **传递函数依赖** | X→Y, Y→Z, 则 X→Z | 学号→系别→系主任 |

## 二、范式递进 ★★★★★

```
┌─ 1NF：每个分量不可再分（最基本的）
│
├─ 2NF：1NF + 消除部分函数依赖
│   → 解决：数据冗余、更新异常
│
├─ 3NF：2NF + 消除传递函数依赖
│   → 解决：传递依赖问题
│
└─ BCNF：3NF + 所有决定因素都是候选码
```

**💡 口诀：** "1NF不可分，2NF消部分，3NF消传递，BCNF更严格"

## 三、做题模板（必会！）

**题目：** R(属性1, 属性2, ...)，FD集 = {...}

**步骤：**
```
① 找候选码
   → 看哪个属性集能推出所有属性

② 判断最高范式
   → 有部分依赖 → 1NF
   → 无部分依赖、有传递依赖 → 2NF
   → 无传递依赖 → 3NF(或BCNF)

③ 分解到3NF
   → 先把部分依赖拆成独立表 → 2NF
   → 再把传递依赖拆成独立表 → 3NF
```

**真题示例：**
```
R(教师号, 课程号, 教师名, 职称, 课程名, 课时, 成绩)

FD: 教师号→教师名,职称；课程号→课程名,课时；(教师号,课程号)→成绩

① 候选码 = (教师号, 课程号)
② 有部分依赖（教师号→教师名），最高 = 1NF
③ 分解：
   教师(教师号, 教师名, 职称)             主码：教师号
   课程(课程号, 课程名, 课时)             主码：课程号
   授课(教师号, 课程号, 成绩)             主码：(教师号,课程号)
                                        外码：教师号→教师, 课程号→课程
```

# 📘 第6章：数据库设计（作图题12分）

## 一、设计6步骤

```
需 → 概 → 逻 → 物 → 实 → 维
（需求分析→概念结构→逻辑结构→物理设计→实施→运行维护）
```

**考试重点：②概念结构（E-R图） + ③逻辑结构（E-R图转关系模型）**

## 二、E-R图表示法

| 图形 | 含义 |
|:----:|:-----|
| **矩形** | 实体 |
| **椭圆** | 属性 |
| **菱形** | 联系（标注1:1 / 1:n / m:n）|
| **线段** | 连接实体与属性/联系 |

## 三、E-R图 → 关系模型 转换规则 ★★★★★

| ER元素 | 转换规则 |
|:-------|:---------|
| **实体** | 每个实体转一张表，属性为表的列 |
| **1:1 联系** | 任一端加另一端主码作为外码 |
| **1:n 联系** | 将"1"端主码加入"n"端作为外码 |
| **m:n 联系** | 单独转为新表，两端主码联合做主码 |
| **联系的属性** | 放在单独为联系创建的表中 |

**💡 记忆口诀：** "实体转表，1:n加外码，m:n拆新表"

## 四、作图题答题模板

**题目：** "一个班级有多个学生，一个学生只属于一个班级……"

**(1) E-R图：**
```
[班级] ──1── 属于 ──n── [学生]
  |                         |
班级号,专业名          学号,姓名,性别
```

**(2) 关系模式：**
```
班级(班级号, 专业名)                      主码：班级号
学生(学号, 姓名, 性别, 班级号)            主码：学号
                                          外码：班级号→班级(班级号)
```

# 📘 第8章：并发控制 + 数据库恢复（分析题18分）

## 一、事务（Transaction）

```sql
BEGIN TRANSACTION
  UPDATE 账户 SET 余额=余额-1000 WHERE 账号='A';
  UPDATE 账户 SET 余额=余额+1000 WHERE 账号='B';
IF 正常 COMMIT;    -- 提交
ELSE ROLLBACK;     -- 回滚
```

### ACID四大特性 ★★★

| 特性 | 含义 | 场景 |
|:----|:-----|:-----|
| **A**原子性 | 全做或全不做 | 转账扣钱+加钱必须同时成功 |
| **C**一致性 | 数据总是正确的 | 转账前后总金额不变 |
| **I**隔离性 | 事务互不干扰 | 你选课不影响我选课 |
| **D**持久性 | 提交了就不丢 | 系统重启也不丢失 |

## 二、并发控制

### 三类数据不一致 ★★★

| 问题 | 现象 | 场景 |
|:----|:-----|:-----|
| **丢失修改** | 后写覆盖前写 | 两人同时改库存，只减了一次 |
| **不可重复读** | 两次读同一数据结果不同 | 读余额两次不一样 |
| **读脏数据** | 读到未提交的数据 | 读到别人还没确认的数据 |

### 封锁技术

| 锁 | 符号 | 功能 | 兼容性 |
|:--|:----:|:-----|:-------|
| **共享锁** | **S锁** | 只读不写 | 多个S锁可共存 |
| **排他锁** | **X锁** | 可读可写 | 独占，与其他锁互斥 |

**三级封锁协议：**
| 级别 | 要求 | 解决 |
|:----|:-----|:------|
| 一级 | 修改前加X锁(事务结束释放) | 防止丢失修改 |
| 二级 | 一级 + 读前加S锁(读完就放) | 防止读脏数据 |
| 三级 | 一级 + 读前加S锁(事务结束释放) | 防止不可重复读 |

### 二段锁协议（2PL）
```
第一阶段：加锁阶段（只加锁，不解锁）
第二阶段：解锁阶段（只解锁，不加锁）
```
> 满足二段锁 → 一定可串行化 ✅

### 并发调度计算模板

**给调度追踪A和B值 → 与顺序执行对比 → 一致则可串行化**

## 三、数据库恢复

### 故障类型

| 故障 | 原因 | 严重程度 |
|:----|:-----|:---------|
| **事务故障** | 单个事务失败（除零等） | ⭐轻 |
| **系统故障** | 系统崩溃、断电 | ⭐⭐中 |
| **介质故障** | 磁盘损坏 | ⭐⭐⭐重 |

### 恢复策略（核心公式）★★★★★

```
有 COMMIT → REDO（重做）
无 COMMIT → UNDO（撤销）
```

- **UNDO：** 根据日志中的"旧值"，恢复原样
- **REDO：** 根据日志中的"新值"，重新执行

### 各类故障恢复

| 故障 | 方法 |
|:-----|:------|
| **事务故障** | 系统自动UNDO（回滚）|
| **系统故障** | UNDO未提交 + REDO已提交 |
| **介质故障** | 还原备份 → REDO已提交 → UNDO未提交 |

### 故障恢复答题模板

```
① 遍历日志，找COMMIT
② 有COMMIT → 标REDO；无COMMIT → 标UNDO
③ 写具体操作：UNDO=写回旧值，REDO=重写新值
```
