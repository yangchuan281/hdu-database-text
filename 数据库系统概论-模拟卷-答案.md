## 二、分析题（37分）

### 1. SQL 改错（6分）

**(1)** ❌ 缺少降序 `DESC`
```sql
SELECT 姓名, 年龄 FROM 学生 ORDER BY 年龄 DESC;
```

**(2)** ❌ 缺少 `GROUP BY`
```sql
SELECT 院系, AVG(年龄) FROM 学生 GROUP BY 院系;
```

**(3)** ❌ `OR` 应为 `AND`，且学号前应加表名前缀
```sql
SELECT 姓名 FROM 学生, 选课 WHERE 学生.学号 = 选课.学号 AND 选课.课程号 = 'C001';
```
💡 经验：WHERE 中连接条件用 AND，筛选条件也用 AND

---

### 2. 规范化（8分）

**(1)** 候选码 = **(教师号, 课程号)**

推导：教师号→教师名,职称；课程号→课程名,课时；(教师号,课程号)→成绩 → 所有属性

**(2)** 最高范式：**1NF**
- 理由：存在部分函数依赖
  - 教师号→教师名,职称（教师号只是候选码的一部分）
  - 课程号→课程名,课时（课程号只是候选码的一部分）
- 部分依赖 → 不满足 2NF → 最高 1NF

**(3)** 分解到 3NF：

| 关系模式 | 主码 | 外码 |
|:---------|:----:|:----:|
| 教师(教师号, 教师名, 职称) | 教师号 | — |
| 课程(课程号, 课程名, 课时) | 课程号 | — |
| 授课(教师号, 课程号, 成绩) | (教师号, 课程号) | 教师号→教师；课程号→课程 |

✅ 全部满足 3NF（也满足 BCNF）

💡 **分解套路：** 先拆部分依赖→再拆传递依赖→标主码外码

### 3. 并发控制（8分）

**(1)** 按调度追踪：
```
T1: READ(A)=1000 → A=800 → WRITE(A)=800
T2: READ(A)=800  → A=960 → WRITE(A)=960
T1: READ(B)=500  → B=700 → WRITE(B)=700
```
最终：**A=960, B=700**

验证——顺序执行 T1→T2：
```
T1: A=800, B=700 → T2: A=800×1.2=960
```
结果一致 → **可串行化调度** ✅

**(2)** 
- **S锁（共享锁）：** 只读不写，多个事务可同时加S锁
- **X锁（排他锁）：** 可读可写，只能一个事务独占
- **二段锁协议：** 第一阶段只加锁不解锁 → 第二阶段只解锁不加锁
- 满足二段锁的调度一定可串行化（充分条件）

💡 **记忆：** S=阅览室（多人看），X=厕所（一人锁门）

---

### 4. 故障恢复（10分）

**(1)**

| 事务 | 核心判断 | 操作 |
|:---:|:--------:|:----:|
| T1 (A:50→80, B:100→150) + COMMIT | ✅ 已提交 | **REDO** |
| T2 (C:200→180) 无 COMMIT | ❌ 未提交 | **UNDO** |
| T3 (D:300→250) 无 COMMIT | ❌ 未提交 | **UNDO** |

**(2)**
- **UNDO：** T2写回 C=200，T3写回 D=300
- **REDO：** T1重写 A=80，B=150

**(3) 介质故障恢复**
- **额外条件：** 必须有数据库的离线备份
- **步骤：** ① 还原最近备份 → ② REDO已提交事务 → ③ UNDO未提交事务
- **与系统故障的区别：** 系统故障只需日志，介质故障需要先还原备份

💡 **必背模板：** "有COMMIT→REDO，无COMMIT→UNDO"

## 三、作图题（12分）

**(1) E-R图：**

```
[读者] ——m—— 借阅(借阅日期,应还日期) ——n—— [图书]
                                              |
                                              n
                                              | 管理
                                              1
                                              |
                                           [管理员]
```
联系类型：借阅 = m:n（多对多），管理 = 1:n（一对多）

**(2) 关系模式：**

| 关系模式 | 主码 | 外码 |
|:---------|:----:|:----:|
| 读者(读者编号, 姓名, 电话, 借阅数量) | 读者编号 | — |
| 图书(图书编号, 书名, 作者, 出版社, 库存量, 工号) | 图书编号 | 工号→管理员 |
| 管理员(工号, 姓名, 联系电话) | 工号 | — |
| 借阅(读者编号, 图书编号, 借阅日期, 应还日期) | (读者编号, 图书编号) | 读者编号→读者；图书编号→图书 |

💡 **转换规则回顾：**
| ER元素 | 转表方式 |
|:-------|:---------|
| 实体 | 直接转为一张表，属性就是列 |
| 1:1 联系 | 任一端加另一端主码作为外码 |
| 1:n 联系 | "1"端主码加入"n"端作为外码 |
| m:n 联系 | 单独转为一张新表，两端主码联合做主码 |
| 联系的属性 | 放在单独为联系创建的表中 |

## 四、编程题（36分）

### 1. SQL 编程（24分）

**(1)** 查询计算机系学生的学号和姓名
```sql
SELECT 学号, 姓名 FROM 学生 WHERE 院系 = 'CS';
```

**(2)** 年龄大于20岁的男学生
```sql
SELECT 姓名, 年龄 FROM 学生 WHERE 年龄 > 20 AND 性别 = '男';
```

**(3)** 选修了C02课程的学生姓名和成绩
```sql
SELECT 学生.姓名, 选课.成绩
FROM 学生, 选课
WHERE 学生.学号 = 选课.学号 AND 选课.课程号 = 'C02';
```

**(4)** 选修了"数据库原理"课程的学生学号和成绩
```sql
SELECT 选课.学号, 选课.成绩
FROM 选课, 课程
WHERE 选课.课程号 = 课程.课程号 AND 课程.课程名 = '数据库原理';
```

**(5)** 各门课程的课程号和选课人数
```sql
SELECT 课程号, COUNT(学号) AS 选课人数
FROM 选课
GROUP BY 课程号;
```

**(6)** 选修课程数≥3门的学生学号
```sql
SELECT 学号 FROM 选课
GROUP BY 学号
HAVING COUNT(课程号) >= 3;
```

**(7)** 创建视图
```sql
CREATE VIEW 计算机系学生 AS
SELECT 学号, 姓名, 年龄 FROM 学生 WHERE 院系 = 'CS';
```

**(8)** 授权给张三（含转授权）
```sql
GRANT SELECT ON 学生 TO 张三 WITH GRANT OPTION;
```

💡 **SQL 模板速记：**
| 场景 | 模板 |
|:-----|:-----|
| 单表查询 | `SELECT 列 FROM 表 WHERE 条件` |
| 多表连接 | `SELECT 列 FROM A, B WHERE A.key=B.key AND 条件` |
| 分组统计 | `SELECT 分组列, 聚合函数 FROM 表 GROUP BY 分组列 HAVING 条件` |
| 创建视图 | `CREATE VIEW 视图名 AS SELECT语句` |
| 授权 | `GRANT 权限 ON 表 TO 用户 [WITH GRANT OPTION]` |
| 收回权限 | `REVOKE 权限 ON 表 FROM 用户` |

### 2. 等价表达（3分）

SQL:
```sql
SELECT 姓名 FROM 学生 WHERE 学号 IN (
  SELECT 学号 FROM 选课 WHERE 课程号 = 'C01'
);
```

等价关系代数：
```
π 姓名( σ 课程号='C01'(学生 ⋈ 选课) )
```
或：
```
π 姓名( 学生 ⋈ (π 学号( σ 课程号='C01'(选课) )) )
```

💡 **对应关系：** SELECT = π，WHERE = σ，JOIN = ⋈

## 🎯 考前5分钟速记卡片

### 📌 三级模式 & 独立性
```
外模式 → 外模式/模式映象(逻辑独立性) → 模式 → 模式/内模式映象(物理独立性) → 内模式
```

### 📌 关系代数 - 基本运算（5个）
**σ（选择行）π（投影列）×（笛卡尔积）∪（并）−（差）**

### 📌 SQL 核心结构
```sql
SELECT 列 FROM 表 WHERE 条件 GROUP BY 列 HAVING 条件 ORDER BY 列;
```

### 📌 ER图 → 关系模式
```
实体→表，1:n→加外码，m:n→拆新表
```

### 📌 范式递进
```
1NF(不可分) → 2NF(消部分依赖) → 3NF(消传递依赖) → BCNF
```

### 📌 ACID
**原一隔久** — 原子性(全做全不做)、一致性(数据正确)、隔离性(互不干扰)、持久性(提交不丢)

### 📌 故障恢复
**有COMMIT → REDO，无COMMIT → UNDO**
**UNDO = 改回旧值，REDO = 重写新值**

### 📌 事务定义
```sql
BEGIN TRANSACTION → 操作序列 → COMMIT(提交) / ROLLBACK(回滚)
```
